# GCFeed - UI/UX 规格（完整功能版）

> 版本：1.0  
> 日期：2026-05-03  
> 产品定位：电脑 Web 端短视频 Feed + 完整运营后台  
> 关联文档：`README.md`、`docs/product.md`、`docs/architecture.md`、`docs/modules/*.md`

## 0. 快速摘要

- 产品一句话：GCFeed 是面向电脑刷短视频用户的 Web Feed 产品，提供 16:9 桌面沉浸浏览、发布、互动、关系、消息、审核、运营、推荐治理、播放优化、监控告警和系统治理能力。
- 前端目标：实现设计文档中的完整功能页面与完整交互状态，覆盖 `docs/product.md` 和 `docs/modules/*.md` 的全部 P0/P1 能力。
- 核心用户：在电脑浏览器里刷抖音/TikTok 式短视频的用户，以及负责审核、运营、监控和治理的后台人员。
- 主设备：电脑 Web，默认 1440px 到 1920px 宽屏，核心舞台按 16:9 设计；手机网页提供窄屏自适应体验。
- 风格关键词：抖音/TikTok 式沉浸、Apple 高级质感、黑白强对比、毛玻璃、精细动效、信息清晰。
- 技术落地：前端先实现完整页面、状态和交互；数据层使用统一适配层，真实接口接入时替换 mock 数据源。

## 0.1 使用方式

- 这份文档是 UI 的唯一真相，颜色、字号、间距、组件、页面结构都以这里为准。
- 新增页面、组件或交互时，先更新本文件对应章节，再进入实现。
- token（设计变量：全站统一的颜色、字号、间距等）来自第 4 节。
- variant（同一组件的不同样式：比如主按钮、危险按钮）来自第 5 节。
- 前端交付以完整体验为准：页面、导航、空状态、加载、错误、成功反馈、权限状态、响应式全部具备。

## 1. 产品范围

### 1.1 完整前端范围

- 账户：注册、登录、退出、当前用户资料、资料编辑、角色状态。
- 关系：关注、取关、关注列表、粉丝列表、作者关系状态。
- 视频：Web 发布视频、视频详情、我的作品、作品状态、删除视频。
- Feed：16:9 桌面刷视频、上下切换、游标分页、刷新、曝光/观看/完播上报。
- 互动：点赞、取消点赞、收藏、取消收藏、评论列表、发表评论、删除评论。
- 消息：消息列表、未读计数、批量已读、点赞/评论/关注/系统消息分类。
- 推荐：候选召回、排序打散、曝光去重、规则配置、效果指标展示。
- 审核：Agent 初审、人审任务、审核详情、通过、驳回、违规下架、审核日志。
- 后台运营：视频查询、审核任务查询、任务分配、运营配置管理、操作日志。
- 播放优化：播放参数、预加载建议、首帧、卡顿、播放质量视图。
- 系统治理：限流检查、降级开关、死信任务重试、治理日志。
- 监控告警：指标看板、告警规则、告警事件、触发中与恢复状态。

### 1.2 产品形态

- 用户 Web 端：面向电脑刷短视频用户，提供沉浸 Feed、发布、互动、关系、消息和个人中心。
- 运营 Web 端：面向运营、审核、管理员，提供审核、内容、推荐、播放、监控、告警、治理、配置。
- 同一个 Web 应用通过角色与导航切换用户端和运营端。

### 1.3 设计原则

1. 电脑刷视频优先：主舞台按 16:9 宽屏设计，鼠标滚轮、方向键、J/K 键都能切视频。
2. 内容第一眼可见：视频、标题、作者、互动数据在首屏形成完整判断。
3. 抖音/TikTok 习惯：关注/推荐频道、快速切换、点赞动效、评论侧栏、作者关注入口保持短视频产品直觉。
4. Apple 高级质感：浅灰背景、黑色沉浸区、毛玻璃面板、细边框、柔和阴影、短弹性动效。
5. 完整状态：每个页面具备加载、空状态、错误、成功、权限受限、提交中。
6. 后台高效率：运营后台以表格、筛选、抽屉详情、批量操作和状态标签为核心。

## 2. 信息架构与导航

### 2.1 用户 Web 端导航

- 顶部栏：GCFeed 标识、关注/推荐频道、全局搜索、发布按钮、消息入口、用户菜单。
- 左侧栏：Feed、发布、消息、我的作品、个人资料、关系。
- Feed 主体：左侧导航 220px，中间 16:9 视频舞台，右侧 360px 评论与详情区。
- 快捷操作：滚轮切换、方向键切换、J/K 切换、L 点赞、F 收藏、C 评论、Space 播放/暂停。

### 2.2 运营 Web 端导航

- 顶部栏：页面标题、全局搜索、环境标识、角色、用户菜单。
- 侧边栏：审核工作台、内容管理、推荐治理、播放优化、指标监控、告警规则、告警事件、系统治理、配置管理、操作日志。
- 详情形态：表格行点击打开右侧抽屉；复杂操作进入详情页；危险动作进入确认弹窗。

## 3. 核心流程

### 3.1 电脑刷 Feed

1. 用户进入 `/feed`，默认频道为推荐。
2. 页面拉取 Feed 数据，16:9 舞台展示当前视频。
3. 竖屏视频在 16:9 舞台内居中播放，左右用封面模糊背景和暗色渐变补齐。
4. 用户滚轮下滑、方向键下、J 键进入下一条；滚轮上滑、方向键上、K 键回上一条。
5. 视频曝光、观看、完播事件自动上报。
6. 用户点赞、收藏、评论、关注作者，UI 即时更新并保留失败回滚能力。
7. 右侧评论区保持可见，评论输入固定在底部。

### 3.2 发布到审核

1. 用户进入发布页，拖拽视频或点击上传。
2. 页面展示本地预览、封面、标题、简介、可见范围。
3. 点击发布，进入上传中与提交中状态。
4. 成功后作品进入“审核中”，用户在我的作品查看。
5. Agent 初审和人工复审结果通过状态标签、消息通知、作品详情展示。

### 3.3 互动与消息

1. 用户点赞、评论、收藏、关注作者。
2. 操作结果即时反馈，后台事件生成站内消息。
3. 消息中心展示点赞、评论、关注、系统通知。
4. 用户可按类型筛选消息，执行批量已读。

### 3.4 审核与内容治理

1. 运营进入审核工作台，按状态、风险分、时间、审核员筛选。
2. 点击任务进入详情抽屉或详情页。
3. 查看 9:16 视频预览、Agent 风险分、作者信息、历史日志。
4. 选择通过或驳回，填写原因或选择原因模板。
5. 对已上线违规内容执行下架，写入操作日志。

### 3.5 推荐、播放、监控、治理

1. 运营查看推荐治理页，调整召回、排序、打散、曝光去重配置。
2. 播放优化页展示首帧、卡顿、预加载、缓冲阈值。
3. 指标监控页展示 Feed 成功率、互动成功率、审核待处理、首帧 P95、告警中数量。
4. 告警规则页创建或启停规则，告警事件页查看触发与恢复。
5. 系统治理页调整降级开关，重试死信任务，查看治理日志。

### 4.1 风格方向

- 用户 Web：黑色沉浸背景、16:9 桌面视频舞台、强对比文字、右侧评论面板、抖音/TikTok 式快速切换。

### 4.5 16:9 Feed 舞台规则

- 外层舞台固定 `aspect-ratio: 16 / 9`，宽度至少 720px，高度随视口变化。
- 竖屏短视频使用 `object-fit: contain` 居中显示。
- 背景层使用当前视频封面，`scale(1.08)`、`blur(28px)`、透明度 55%。
- 舞台左右使用暗色渐变承接背景，保证文字与按钮可读。
- 右侧互动栏贴近视频右缘，评论区占 360px。
- 桌面滚轮一次切一条视频，动画时长 360ms。
- 窄屏切换为单列 9:16 浏览体验。

## 6. 页面规格

### 6.1 登录/注册

- 结构：居中表单面板、账号、密码、昵称、登录、注册。
- 状态：输入校验、提交中、登录成功、账号冲突、密码错误、权限状态。
- 跳转：普通用户进入 Feed；运营角色进入审核工作台。

### 6.2 Feed 浏览

- 结构：顶部导航、左侧栏、16:9 FeedStage、互动栏、右侧评论/详情区。
- 主操作：切换下一条视频。
- 必备能力：关注/推荐频道、搜索入口、播放/暂停、滚轮切换、键盘切换、点赞、收藏、评论、分享、关注作者、曝光上报、观看上报、完播上报。
- 桌面布局：220px 左侧栏 + minmax(720px, 1fr) 舞台 + 360px 评论区。
- 窄屏布局：单列 9:16，全屏浏览，底部导航。
- 状态：首屏骨架、视频加载、播放错误、Feed 空状态、接口错误、操作成功、操作回滚。

### 6.3 视频详情

- 结构：16:9 详情舞台、作者信息、互动数据、评论、相关推荐。
- 能力：播放、点赞、收藏、评论、关注作者、进入作者作品、查看推荐来源。
- 状态：详情加载、评论加载、空评论、视频下架、视频删除。

### 6.4 发布视频

- 结构：上传区、视频预览、封面、标题、简介、可见范围、发布按钮。
- 能力：拖拽上传、封面选择、标题校验、上传进度、提交审核、发布成功跳转我的作品。
- 状态：空上传、上传中、上传失败、提交中、审核中。

### 6.5 我的作品

- 结构：资料摘要、状态 Tab、作品网格、作品操作。

- 状态：作品骨架、空作品、删除确认、操作成功。

### 6.6 消息中心

- 结构：消息分类 Tab、未读计数、消息列表、批量已读。

- 状态：空消息、加载、已读成功。

### 6.7 个人资料

- 结构：头像、昵称、简介、账号状态、保存按钮。

- 状态：保存中、保存成功、校验错误。

### 6.8 关系页

- 结构：关注列表、粉丝列表、搜索作者、关系状态。

- 状态：空关注、空粉丝、操作成功。

### 6.9 搜索结果

- 结构：搜索框、视频结果、作者结果、标签/话题结果。
- 能力：关键词搜索、筛选、进入视频、关注作者。
- 状态：搜索中、空结果、搜索错误。

### 6.10 审核工作台

- 结构：状态指标卡、筛选栏、任务表格、批量操作、详情抽屉。
- 能力：状态筛选、风险分筛选、时间筛选、审核员筛选、任务分配、通过、驳回。
- 状态：表格骨架、空任务、操作成功、冲突提示。

### 6.11 审核详情

- 结构：9:16 预览、标题、作者、Agent 风险分、审核日志、判定区。
- 能力：播放预览、查看风险原因、通过、驳回、下架、查看操作记录。
- 状态：加载、视频不可用、提交中、提交成功。

### 6.12 内容管理

- 结构：筛选栏、视频表格、详情抽屉、下架弹窗。
- 能力：按标题、作者、状态、发布时间、互动数据筛选；查看；下架；删除。
- 状态：空列表、加载、下架成功。

### 6.13 推荐治理

- 结构：策略概览、召回配置、排序配置、打散配置、曝光去重指标。
- 能力：启停规则、编辑参数、保存、查看效果曲线。
- 状态：保存中、保存成功、配置校验错误。

### 6.14 播放优化

- 结构：播放参数、预加载建议、首帧趋势、卡顿趋势、QoS 日志。
- 能力：按平台和网络筛选、调整预加载条数、调整缓冲阈值、查看异常视频。
- 状态：指标加载、空指标、保存成功。

### 6.15 指标监控

- 结构：时间范围、核心指标卡、趋势图、异常列表。
- 指标：Feed 拉取成功率、首帧 P95、互动写入成功率、审核待处理数、告警中数量。
- 状态：图表骨架、空数据、刷新成功。

### 6.16 告警规则

- 结构：规则列表、创建规则表单、启停开关。
- 能力：创建、编辑、启停、删除规则。
- 状态：保存成功、表达式校验错误。

### 6.17 告警事件

- 结构：状态 Tab、事件表格、详情抽屉。
- 能力：查看触发中、已恢复、触发值、恢复时间、关联指标。
- 状态：空告警、加载、刷新成功。

### 6.18 系统治理

- 结构：降级开关、限流状态、死信任务、治理日志。
- 能力：调整降级开关、重试死信任务、查看最后错误。
- 状态：保存成功、重试成功、重试失败。

### 6.19 配置管理

- 结构：配置分类、配置项表格、JSON 编辑区、操作日志。
- 能力：编辑运营配置、保存、回滚、查看更新人。
- 状态：JSON 校验错误、保存成功、回滚成功。

### 6.20 操作日志

- 结构：操作人、动作类型、目标类型、目标 ID、详情、时间。
- 能力：按人、动作、目标、时间筛选；查看 JSON 详情。
- 状态：空日志、加载、查询错误。

## 7. 文案规范

- 整体语气：短、轻、直接。
- Feed 操作文案：关注、已关注、点赞、收藏、评论、分享、下一条。
- 发布文案：发布视频、上传中、发布成功，已进入审核。
- 审核文案：通过、驳回、下架、填写原因、审核结果已提交。
- 监控文案：刷新数据、创建规则、告警中、已恢复。
- 错误模板：`加载失败，请重试`、`保存失败，请检查输入`、`当前账号需要运营权限`。

## 8. 可访问性与快捷键

- 所有按钮、输入框、Tab、表格行操作支持键盘聚焦。
- Feed 快捷键：`J` 下一条、`K` 上一条、`L` 点赞、`F` 收藏、`C` 聚焦评论、`Space` 播放/暂停。
- 焦点样式使用 Apple 蓝外环。
- Feed 舞台文字有遮罩，评论区和表格满足清晰对比。
- 点击热区：Feed 图标按钮 52px，后台行操作 32px 以上。
